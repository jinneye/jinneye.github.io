春秋小记1

发表于 2020-05-31

5月份读了《春秋谷梁传》，每次读大部头的书都会发现一次自己的认知误区。这次也不例外，写一些小的摘抄和随感。

这次读的是上海古籍出版社《十三经译注》中的《春秋谷梁传 译注》，承载撰，书中内容分 经、传、注释和译文四部分，经传是古文看不大懂，主要看注释和译文。
好书的前言部分也很有价值，这本《春秋谷梁传》也不例外。前言中比较了《公羊传》、《左传》、《谷梁传》三传的异同，《公羊》和《谷梁》都是采用问答式的文体来作解释，内容侧重于解说公文的意义，史实部分相对简略。左传祥于记事，不仅丰富了书中的内容，还有所补充，所以读左传可以更多的了解历史。三传对于同一条经文的解说却有可能褒贬不一，也印证了“尽信书，不如无书”的观点。
西汉时期经学的发展，及受政治局势的左右。“罢黜百家，独尊儒术”也是汉武帝为了维护统治做出来的决策，却影响了中国人上千年的思想，上有好焉，下必甚焉，中国文人的终极目标还是“货与帝王家”，帝王注意到了儒家学说有利于统治的一面，颁布相关政策，儒家学说就迅速发展起来了。和现代国家出一个规划，学界就纷纷研究相关课题何其相似，中国估计也只有在春秋战国时期有过学术独立，秦汉之后就都只是为皇权服务。前言中还提到宣帝不喜“缘经饰事”的儒术，但是很看重礼乐制度，可见帝王的政治洞察力远胜于一般的文人墨客。

第一个小故事是鲁国文化的起源。周朝建立后，开始分封功臣。当时为了监视商朝后裔和东方的夷人（齐鲁大地原是商朝遗民居住的地方，再往南边沿海的地方就是九夷，后来属于吴，再后来属于越，再后来就属于楚，“楚虽三户，亡秦必楚”的那个楚，楚文化也一直不属于中原文化），周公就把东方的齐鲁两地分别分封给了开国功臣姜太公和周公，鲁国由周公的儿子代为执政。周公和姜太公聊起执政方针，太公问：“何以治鲁”，周公曰“尊尊而亲亲”，太公曰“后世浸（逐渐）弱矣”；周公曰“何以治齐”，姜太公曰“举贤而上功”，周公曰“后世必有篡杀之臣”。姜太公自己本身就是夷人，对商遗民和夷人比较宽松，采用“因其俗，简其理”的政策，大胆任用人才，虽然分封之初齐国不如鲁国，后期国力迅速强盛，齐桓公时期成为春秋第一位霸主，战国时期齐威王更是创立了天下闻名的“稷下学宫”，收纳了众多学者，形成了不同的学派和思想体系，成为当时百家争鸣的思想交流中心。而鲁国遵从“尊尊亲亲”，“启以商政，疆以周索”，严格按周礼来治理国家，崇尚先王之训，大讲礼乐教化，成为当时有名的礼仪之邦，当时各国都要到鲁国来学习周礼，但是有才华的人很难得到任用，即便是孔子这种提倡礼教的儒家在鲁国也难有发展前途。最终齐鲁两国的发展都如周公、太公所预言，鲁国日渐衰弱，东周时期就要看齐国脸色，公元前249被楚国灭亡；而齐国于公元前379年发生了“田氏代齐”被家臣篡位，于221被秦国灭亡。对当时的老百姓，到底是生活在齐国比较幸福呢还是鲁国比较幸福呢？

开通博客用到的一些指令

发表于 2020-05-31

开通博客让自己重温了用linux指令做事的风格，已经离开linux界面十来年，对于用键盘输入指令做事而不是通过鼠标已经很不熟悉了，故将这次用到的指令记下来，以备不时查询。

创建过程

1、创建Github 域名和空间
按照官网指示

2、安装Git, Nodejs, Hexo
检查是否安装Git
$ git –version
如果没有出现版本号就需要安装，win10直接下载Git软件安装就可以
https://git-scm.com/downloads
nodejs
win10在官网下载安装 https://nodejs.org/zh-cn/
安装好后用CMD命令检查是否成功
node -v 检查node版本
npm -v 检查npm版本
hexo
安装hexo
$ npm install -g hexo-cli
检查hexo是否正确安装上
$ hexo –version

3、创建项目
$ hexo init 项目名称，比如项目名称是test 那就是
$ hexo init test，用hexo初始化test目录，生成相关配置。

4、安装主题
$ cd test 进入项目目录
在https://hexo.io/themes/上有一部分免费主题，clone了下面这个很受欢迎的next主题。
$ git clone https://github.com/theme-next/hexo-theme-next

5、基础配置和主题配置
基础配置在主目录下面的_config.yml文件中设置
$ vi _config.yml进入编辑页面，设置主题配置，根据五分钟那篇博主的推荐改了下面几个值
title: ** //你博客的名字
author: ** //你的名字
language: ** //语言 中文 （这个要去看next主题目录下面语言的设置，版本不一样文件名不一样）
theme: next //刚刚安装的主题名称
deploy:
type: git //使用Git 发布
repo: https://github.com/username/username.github.io.git // 刚创建的Github仓库
主题配置在next目录下的_config.yml中设置
$ cd username.github.io/themes/next/
$ vi _config.yml

6、写文章
在主目录下创建新的md文件
$ hexo new post 文件名
可直接在/source/_posts文件夹下面创建md文件，再用markdown打开就可以编写。
这里最早用了“hexo new “文件名”的指令，结果就直接进入 > 模式，不知道怎么退出来，用了esc和 ：q! 都不行，最后都是直接关git bash窗口强制退出。

7、测试
$ hexo s
在浏览器中输入https://localhost:4000访问静态页面
用”ctrl+C”退出测试模式

8、自动发布
$ npm install hexo-deployer-git –save 安装hexo-deployer-git自动发布工具
$ hexo clean #清楚之前生成的东西，可用可不用
$ hexo g #生成静态文章
$ hexo d #部署文章
也可以直接用
$ hexo clean && hexo g && hexo d

9、设置SSH
看网上的教程这一步是在前面的，我是在发布的时候才发现自己没有设置这步。参考网上另一个教程，做了ssh设置。
在Git Bash输入以下指令，检查是否已经存在了SSH keys。
ls -al ~/.ssh
如果存在就删除.ssh文件夹所有文件
输入github的用户名和注册邮箱
$ git config –global user.name “yourname”
$ git config –global user.email “youremail”
检查是否正确
$ git config user.name
$ git config user.email
创建SSH
$ ssh-keygen -t rsa -C “youremail”
$ clip < ~/.ssh/id_rsa.pub 复制rsa.pub的信息
在GitHub的setting中，找到SSH keys的设置选项，点击New SSH key
把id_rsa.pub里面的信息复制进去
检查是否成功
ssh -T git@github.com

10、浏览器查看：http://username.github.io/

hexo指令

用于生成静态文件
hexo generate
hexo g #简写
情况pubilc文件
hexo clean
启动服务预览
hexo s
hexo server
部署站点，本地生成.deploy_git文件。并建编译后的文件上传至githab
新建文章
hexo new post title
一键部署
hexo clean && hexo generate && hexo deploy

linux常用指令

文件和目录
cd /home 进入 ‘/ home’ 目录’
cd .. 返回上一级目录
cd ../.. 返回上两级目录
cd 进入个人的主目录
cd - 返回上次所在的目录
pwd 显示工作路径
ls 查看目录中的文件
ls -F 查看目录中的文件
ls -l 显示文件和目录的详细资料
ls -a 显示隐藏文件
ls [0-9] 显示包含数字的文件名和目录名
tree 显示文件和目录由根目录开始的树形结构
lstree 显示文件和目录由根目录开始的树形结构
mkdir dir1 创建一个叫做 ‘dir1’ 的目录’
mkdir dir1 dir2 同时创建两个目录
mkdir -p /tmp/dir1/dir2 创建一个目录树
rm -f file1 删除一个叫做 ‘file1’ 的文件’
rmdir dir1 删除一个叫做 ‘dir1’ 的目录’
rm -rf dir1 删除一个叫做 ‘dir1’ 的目录并同时删除其内容
rm -rf dir1 dir2 同时删除两个目录及它们的内容
mv dir1 new_dir 重命名/移动 一个目录
cp file1 file2 复制一个文件
cp dir/* . 复制一个目录下的所有文件到当前工作目录
cp -a /tmp/dir1 . 复制一个目录到当前工作目录
cp -a dir1 dir2 复制一个目录
cp -r dir1 dir2 复制一个目录及子目录

搭建博客遇到的问题

发表于 2020-05-31

一直想着建自己的博客，某天在网上看到了这一篇“5分钟搭建免费个人博客”（https://www.jianshu.com/p/4eaddcbe4d12),
然后就心动入坑，结果花了三天才搞定。
在github上用hexo搭建博客的攻略网上很多，基本上按照简书这位的步骤就可以了。
1、创建Github域名和空间，这一步比较简单，按照github的指南去做就好。
2、安装Git、Nodjes、hexo。安装命令网上都有，win10可以通过下载安装程序来安装，或者通过官网指导安装。
3、编写发布。
这个环节因为不熟悉git bash指令，有个小问题折腾了三个晚上，在用$ hexo s测试静态网页的时候，无论如何都打不开
https://localhost:4000，
按照网上各种攻略改端口，改网页设置，折腾到最后发现原来每次我都是发布$ hexo s之后就顺手用“ctrl+c” 退出测试了，自然怎么改都看不到静态页面了。
4、Markdown编写博文。到了这里还要下载一个软件已经不耐烦了，作为一个非编程人员，向来是指望下载一个工具就能搞定一件事，因为每件工具都是需要时间精力去学习使用的。现在还要下载第四个，但是已经到了这一步，还是试试吧，markdown的使用目前还不熟练。
5、动态发布。这篇博文里少了设置 ssh key 这一步，网上重新找了一篇攻略完成key的设置，基本都是照着攻略输指令。但是动态发布后github更新有延迟，无法立即刷新网站，这个也是试了一个晚上才发现的，刚开始还以为自己指令输错了。
6、新建文档。可能是版本问题，或者是自己对linux命令不熟悉，用“hexo new 文件名” 没法新建文档，后来改成“hexo new post 文件名”就好了。
7、更改主题。更改主题的时候发现了语言的问题，后来发现是版本的设置不一样，根目录下面的_config.yml语言设置需要和主题里面的语言文件同名才行。另外主题目录下面的_config.yml也可以做一些设定。
8、更多的还是要慢慢探索，很多linux命令已经忘了，重新捡起来。
